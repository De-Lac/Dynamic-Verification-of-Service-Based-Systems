ServicePot: An Extensible Registry for Choreography Governance

This is a sample implementation for ServicePot. It comes preloaded with the Incident Management example from the OMG BPMN 2.0 by Example book. Following are the SOAP messages used to create and retrieve the Choreography.

Input:

        
 <soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/" xmlns:chor="http://www.choreos.eu">
   <soapenv:Header/>
   <soapenv:Body>
      <chor:SaveChoreography>
         <arg0>
            <authInfo>authtoken:83c85311-cb9a-4820-bcac-427037dfb6c5</authInfo>
            <choreographyURI>http://www.omg.org/spec/BPMN/20100602/2010-06-03/Incident%20Management/Incident%20Management%20-%20coll%20chor.bpmn</choreographyURI>
         </arg0>
      </chor:SaveChoreography>
   </soapenv:Body>
</soapenv:Envelope>
